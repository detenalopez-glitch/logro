# Gestión del Proyecto

## Metodología
En este proyecto aplicaremos principios **Agile**, específicamente una mezcla enfocada en **Kanban** para mantener un flujo continuo de trabajo y entregas rápidas.

## Organización del Tablero Trello
Usaremos un tablero de Trello con las siguientes columnas para visualizar el flujo:

1. **Backlog:** Ideas, requerimientos y tareas futuras que aún no están listas para trabajarse.
2. **To Do:** Tareas priorizadas listas para el desarrollo.
3. **In Progress:** Tareas que están siendo trabajadas actualmente. (Regla: no tener más de 2-3 tareas aquí a la vez).
4. **Review:** Tareas finalizadas en código pero pendientes de revisión.
5. **Done:** Tareas completamente terminadas e integradas al proyecto.

## Convenciones de Trabajo
- Las ramas de Git deben nombrarse de manera descriptiva (`feat/nombre-tarea`, `fix/nombre-bug`, `docs/nombre-doc`).
- Usaremos **Conventional Commits** para mantener un historial limpio (ej. `feat: agregar vista de logros`).
- Cada nueva funcionalidad importante debe estar respaldada por su respectiva documentación.
- **Actualización:** El tablero de Trello debe reflejar la realidad en todo momento (actualizar en cada sesión).
